# Git Workflow & Deployment Guide / бһӣбҹҶбһ бһјбһҡбһҖбһ¶бһҡбһ„бһ¶бһҡ Git & бһҖбһ¶бһҡбһҠбһ¶бһҖбҹӢбһ–бһ„бҹ’бһҡбһ¶бһҷ

If you are new to Git and team collaboration, this guide explains exactly how to safely add your code to the main project without breaking anything.  
бһ”бҹ’бһҡбһҹбһ·бһ“бһ”бһҫбһўбҹ’бһ“бһҖбһҗбҹ’бһҳбһёбһ…бҹҶбһ–бҹ„бҹҮ Git бһ“бһ·бһ„бһҖбһ¶бһҡбһҹбһ бһҖбһ¶бһҡбһҮбһ¶бһҖбҹ’бһҡбһ»бһҳ бһҳбһӮбҹ’бһӮбһ»бһ‘бҹ’бһ‘бҹҒбһҹбһҖбҹҚбһ“бҹҒбҹҮбһ–бһ“бҹ’бһҷбһӣбҹӢбһҷбҹүбһ¶бһ„бһ…бҹ’бһ”бһ¶бһҹбҹӢбһ–бһёбһҡбһ”бҹҖбһ”бһ”бһ“бҹ’бһҗбҹӮбһҳбһҖбһјбһҠбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһ‘бҹ…бһҖбҹ’бһ“бһ»бһ„бһӮбһҳбҹ’бһҡбҹ„бһ„бһ…бһҳбҹ’бһ”бһ„бһҠбҹ„бһҷбһҹбһ»бһңбһҸбҹ’бһҗбһ·бһ—бһ¶бһ– бһҠбҹ„бһҷбһҳбһ·бһ“бһ’бҹ’бһңбһҫбһІбҹ’бһҷбһҒбһјбһ…бһўбҹ’бһңбһёбһ‘бһ¶бҹҶбһ„бһўбһҹбҹӢбҹ”

---

## 1. The Rule of the Branches / бһ…бҹ’бһ”бһ¶бһ”бҹӢбһ“бҹғбһҳбҹӮбһҖбһ’бһ¶бһ„ (Branches)
Think of branches as parallel universes.  
бһ…бһјбһҡбһ…бһ¶бһҸбҹӢбһ‘бһ»бһҖбһҳбҹӮбһҖбһ’бһ¶бһ„ (Branches) бһҠбһјбһ…бһҮбһ¶бһҹбһҖбһӣбһӣбҹ„бһҖбһҹбҹ’бһҡбһ”бһӮбҹ’бһ“бһ¶бҹ”

- **`main`**: This is the LIVE universe. This is what users see on the internet. **NEVER edit code directly here.** (бһ“бҹҒбҹҮбһҮбһ¶бһҹбһҖбһӣбһӣбҹ„бһҖбһ•бҹ’бһҹбһ¶бһҷбһ•бҹ’бһ‘бһ¶бһӣбҹӢбҹ” бһңбһ¶бһҮбһ¶бһўбҹ’бһңбһёбһҠбҹӮбһӣбһўбҹ’бһ“бһҖбһ”бҹ’бһҡбһҫбһ”бҹ’бһҡбһ¶бһҹбҹӢбһҳбһҫбһӣбһғбһҫбһүбһ“бҹ…бһӣбһҫбһўбҹҠбһёбһ“бһ’бһәбһҺбһ·бһҸбҹ” **бһ бһ¶бһҳбһҖбҹӮбһ”бҹ’бһҡбҹӮбһҖбһјбһҠбһҠбҹ„бһҷбһ•бҹ’бһ‘бһ¶бһӣбҹӢбһ“бҹ…бһ‘бһёбһ“бҹҒбҹҮбһҮбһ¶бһҠбһ¶бһ…бҹӢбһҒбһ¶бһҸбҹ”**)
- **`development`**: This is the TEST universe. Developers merge their finished work here to see if it plays nicely with other developers' work. (бһ“бҹҒбҹҮбһҮбһ¶бһҹбһҖбһӣбһӣбҹ„бһҖбһҹбһ¶бһҖбһӣбҹ’бһ”бһ„бҹ” бһўбҹ’бһ“бһҖбһўбһ—бһ·бһңбһҢбҹ’бһҚбһ“бҹҚбһ”бһүбҹ’бһҮбһјбһӣбһҖбһ¶бһҡбһ„бһ¶бһҡбһҠбҹӮбһӣбһ”бһ¶бһ“бһ”бһүбҹ’бһ…бһ”бҹӢбһ“бҹ…бһ‘бһёбһ“бҹҒбҹҮ бһҠбһҫбһҳбҹ’бһ”бһёбһҳбһҫбһӣбһҗбһ¶бһҸбһҫбһңбһ¶бһҠбҹҶбһҺбһҫбһҡбһҖбһ¶бһҡбһӣбҹ’бһўбһҮбһ¶бһҳбһҪбһҷбһ“бһ№бһ„бһҖбһ¶бһҡбһ„бһ¶бһҡбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһҠбһ‘бҹғбһ¬бһўбһҸбҹӢбҹ”)
- **`feature/...`**: This is YOUR personal universe. You create this branch to work on your specific task without bothering anyone else. (бһ“бҹҒбҹҮбһҮбһ¶бһҹбһҖбһӣбһӣбҹ„бһҖбһ•бҹ’бһ‘бһ¶бһӣбҹӢбһҒбҹ’бһӣбһҪбһ“бһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбҹ” бһўбҹ’бһ“бһҖбһ”бһ„бҹ’бһҖбһҫбһҸбһҳбҹӮбһҖбһ’бһ¶бһ„бһ“бҹҒбҹҮбһҠбһҫбһҳбҹ’бһ”бһёбһ’бҹ’бһңбһҫбһҖбһ¶бһҡбһ„бһ¶бһҡбһҮбһ¶бһҖбҹӢбһӣбһ¶бһҖбҹӢбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖ бһҠбҹ„бһҷбһҳбһ·бһ“бһҡбҹҶбһҒбһ¶бһ“бһўбҹ’бһ“бһҖбһҠбһ‘бҹғбҹ”)

## 2. Step-by-Step: Writing & Pushing Code / бһҳбһҪбһҷбһҮбҹҶбһ бһ¶бһ“бһҳбҹ’бһҸбһ„бҹ—бҹ– бһҖбһ¶бһҡбһҹбһҡбһҹбҹҒбһҡ бһ“бһ·бһ„бһҡбһ»бһүбһҖбһјбһҠ
Here is the exact process you should follow every time you start a new task:  
бһ“бҹҒбҹҮбһӮбһәбһҮбһ¶бһ“бһёбһҸбһ·бһңбһ·бһ’бһёбһ…бҹ’бһ”бһ¶бһҹбҹӢбһӣбһ¶бһҹбҹӢбһҠбҹӮбһӣбһўбҹ’бһ“бһҖбһӮбһҪбһҡбһўбһ“бһ»бһңбһҸбҹ’бһҸбһҸбһ¶бһҳ бһҡбһ¶бһӣбҹӢбһ–бҹҒбһӣбһ…бһ¶бһ”бҹӢбһ•бҹ’бһҸбһҫбһҳбһҖбһ¶бһҡбһ„бһ¶бһҡбһҗбҹ’бһҳбһёбҹ–

**Step 1: Get the latest code from `development`** (бһ‘бһ¶бһүбһҷбһҖбһҖбһјбһҠбһҗбҹ’бһҳбһёбһ”бҹҶбһ•бһ»бһҸбһ–бһё `development`)
```bash
git checkout development
git pull origin development
```

**Step 2: Create your own personal branch** (бһ”бһ„бҹ’бһҖбһҫбһҸбһҳбҹӮбһҖбһ’бһ¶бһ„бһ•бҹ’бһ‘бһ¶бһӣбҹӢбһҒбҹ’бһӣбһҪбһ“бһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖ)
```bash
git checkout -b feature/my-new-button
```

**Step 3: Write your code, and save your progress** (бһҹбһҡбһҹбҹҒбһҡбһҖбһјбһҠбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖ бһ бһҫбһҷбһҡбһҖбҹ’бһҹбһ¶бһ‘бһ»бһҖбһӣбһ‘бҹ’бһ’бһ•бһӣбһҖбһ¶бһҡбһ„бһ¶бһҡ)
```bash
git add .
git commit -m "feat: added a shiny new button"
```

**Step 4: Push your branch to GitHub** (бһҡбһ»бһүбһҳбҹӮбһҖбһ’бһ¶бһ„бһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһ‘бҹ…бһҖбһ¶бһ“бҹӢ GitHub)
```bash
git push origin feature/my-new-button
```

## 3. Pull Requests (PR) / бһҹбҹҶбһҺбһҫбһ‘бһ¶бһүбһҷбһҖ (Pull Requests)
Once your code is on GitHub, you need to ask the team for permission to merge your personal universe (`feature/my-new-button`) into the shared universe (`development`).  
You do this by clicking the **"Compare & pull request"** button on GitHub.com.  
бһ“бҹ…бһ–бҹҒбһӣбһҖбһјбһҠбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһ“бҹ…бһӣбһҫ GitHub бһўбҹ’бһ“бһҖбһҸбҹ’бһҡбһјбһңбһҹбһ»бҹҶбһҖбһ¶бһҡбһўбһ“бһ»бһүбҹ’бһүбһ¶бһҸбһ–бһёбһҖбҹ’бһҡбһ»бһҳ бһҠбһҫбһҳбҹ’бһ”бһёбһ”бһүбҹ’бһ…бһјбһӣбһҹбһҖбһӣбһӣбҹ„бһҖбһ•бҹ’бһ‘бһ¶бһӣбҹӢбһҒбҹ’бһӣбһҪбһ“бһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖ бһ‘бҹ…бһҖбҹ’бһ“бһ»бһ„бһҹбһҖбһӣбһӣбҹ„бһҖбһҡбһҪбһҳбҹ” бһўбҹ’бһ“бһҖбһ’бҹ’бһңбһҫбһ”бҹӮбһ”бһ“бҹҒбҹҮбһҠбҹ„бһҷбһ…бһ»бһ…бһ”бҹҠбһјбһҸбһ»бһ„ **"Compare & pull request"** бһ“бҹ…бһӣбһҫбһӮбҹҒбһ бһ‘бҹҶбһ–бҹҗбһҡ GitHub.comбҹ”

### The CI Gatekeeper (бһўбҹ’бһ“бһҖбһҷбһ¶бһҳбһ‘бҹ’бһңбһ¶бһҡ CI)
When you open a PR, you must wait about 2 minutes. GitHub Actions (our CI server) will automatically download your code and run `npm run test:unit` and `npm run test:e2e`.  
If you broke something, the PR will turn **RED** and you cannot merge it. If it turns **GREEN**, another developer will review your code and click "Merge"!  
бһ“бҹ…бһ–бҹҒбһӣбһўбҹ’бһ“бһҖбһ”бһҫбһҖ PR бһўбҹ’бһ“бһҖбһҸбҹ’бһҡбһјбһңбһҡбһ„бҹӢбһ…бһ¶бҹҶбһ”бҹ’бһҡбһ бҹӮбһӣ бҹў бһ“бһ¶бһ‘бһёбҹ” GitHub Actions бһ“бһ№бһ„бһ‘бһ¶бһүбһҷбһҖбһҖбһјбһҠбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһҠбҹ„бһҷбһҹбҹ’бһңбҹҗбһҷбһ”бҹ’бһҡбһңбһҸбҹ’бһҸбһ· бһ бһҫбһҷбһҠбҹҶбһҺбһҫбһҡбһҖбһ¶бһҡбһҖбһ¶бһҡбһҹбһ¶бһҖбһӣбҹ’бһ”бһ„бҹ” бһ”бҹ’бһҡбһҹбһ·бһ“бһ”бһҫбһңбһ¶бһӣбҹ„бһҸбһ–бһҺбҹҢ **бһҖбҹ’бһҡбһ бһҳ** бһўбҹ’бһ“бһҖбһҳбһ·бһ“бһўбһ¶бһ…бһ”бһүбҹ’бһҮбһјбһӣбһңбһ¶бһ”бһ¶бһ“бһ‘бҹҒбҹ” бһ”бҹ’бһҡбһҹбһ·бһ“бһ”бһҫбһңбһ¶бһӣбҹ„бһҸбһ–бһҺбҹҢ **бһ”бҹғбһҸбһ„** бһўбҹ’бһ“бһҖбһ•бҹ’бһҹбҹҒбһ„бһўбһ¶бһ…бһ–бһ·бһ“бһ·бһҸбҹ’бһҷ бһ“бһ·бһ„бһ…бһ»бһ…бһ”бһүбҹ’бһҮбһјбһӣ (Merge) бһ”бһ¶бһ“!

## 4. Automatic Deployment (CD) / бһҖбһ¶бһҡбһҠбһ¶бһҖбҹӢбһ–бһ„бҹ’бһҡбһ¶бһҷбһҹбҹ’бһңбҹҗбһҷбһ”бҹ’бһҡбһңбһҸбҹ’бһҸбһ· (CD)
How does the website actually update for the public?  
бһҸбһҫбһӮбҹҒбһ бһ‘бҹҶбһ–бҹҗбһҡбһ’бҹ’бһңбһҫбһ”бһ…бҹ’бһ…бһ»бһ”бҹ’бһ”бһ“бҹ’бһ“бһ—бһ¶бһ–бһҹбһҳбҹ’бһҡбһ¶бһ”бҹӢбһҹбһ¶бһ’бһ¶бһҡбһҺбһҮбһ“бһҠбҹ„бһҷбһҡбһ”бҹҖбһ”бһҺбһ¶?

We use a process called Continuous Deployment (CD). When the team decides that `development` is ready for the public, they will open a PR from `development` into `main`.  
бһҷбһҫбһ„бһ”бҹ’бһҡбһҫбһ”бҹ’бһҡбһ¶бһҹбҹӢбһҠбҹҶбһҺбһҫбһҡбһҖбһ¶бһҡбһ бҹ…бһҗбһ¶ бһҖбһ¶бһҡбһҠбһ¶бһҖбҹӢбһ–бһ„бҹ’бһҡбһ¶бһҷбһҮбһ¶бһ”бһ“бҹ’бһҸбһ”бһ“бҹ’бһ‘бһ¶бһ”бҹӢ (CD)бҹ” бһ“бҹ…бһ–бҹҒбһӣбһҖбҹ’бһҡбһ»бһҳбһҹбһҳбҹ’бһҡбҹҒбһ…бһ…бһ·бһҸбҹ’бһҸбһҗбһ¶ `development` бһҸбҹ’бһҡбҹҖбһҳбһҒбҹ’бһӣбһҪбһ“бһҡбһҪбһ…бһҡбһ¶бһӣбҹӢ бһ–бһҪбһҖбһӮбҹҒбһ“бһ№бһ„бһ”бһҫбһҖ PR бһ–бһё `development` бһ‘бҹ…бһҖбһ¶бһ“бҹӢ `main`бҹ”

The exact moment code touches the `main` branch, our `.github/workflows/deploy.yml` robot wakes up. It builds the `dist/` folder and uploads it to GitHub Pages. Within 60 seconds, your code is live for the whole world to see!  
бһ“бҹ…бһңбһ·бһ“бһ¶бһ‘бһёбһҠбҹӮбһӣбһҖбһјбһҠбһ…бһјбһӣбһҠбһӣбҹӢбһҳбҹӮбһҖбһ’бһ¶бһ„ `main` бһҳбһ“бһ»бһҹбҹ’бһҹбһҷбһ“бҹ’бһҸбһҡбһ”бһҹбҹӢбһҷбһҫбһ„бһ“бһ№бһ„бһ—бҹ’бһүбһ¶бһҖбҹӢбһЎбһҫбһ„ бһ”бһ„бҹ’бһҖбһҫбһҸбһҗбһҸ `dist/` бһ бһҫбһҷбһ”бһ„бҹ’бһ бҹ„бҹҮбһңбһ¶бһ‘бҹ…бһҖбһ¶бһ“бҹӢ GitHub Pagesбҹ” бһҖбҹ’бһ“бһ»бһ„бһҡбһҷбҹҲбһ–бҹҒбһӣ бҹҰбҹ  бһңбһ·бһ“бһ¶бһ‘бһё бһҖбһјбһҠбһҡбһ”бһҹбҹӢбһўбҹ’бһ“бһҖбһ“бһ№бһ„бһҳбһ¶бһ“бһ•бҹ’бһҹбһ¶бһҷбһ”бһ“бҹ’бһҸбһ•бҹ’бһ‘бһ¶бһӣбҹӢбһҹбһҳбҹ’бһҡбһ¶бһ”бҹӢбһ–бһ·бһ—бһ–бһӣбҹ„бһҖбһ‘бһ¶бҹҶбһ„бһҳбһјбһӣ!
